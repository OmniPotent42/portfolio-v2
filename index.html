The world of development is a fast-paced, always changing environment. Frameworks come and go, languages go in and out of style, and applications get more and more complex.

With each new architecture comes advancements in performance and efficiency. The last thing anyone wants is to get stuck with an outdated application that can't keep up with today's demands.

Why Serverless?

Serverless is the latest back-end architecture, born out of services like AWS Lambda. This allows for highly scalable and low cost applications that can be run on large server clusters, like the ones provided by Amazon Web Services. The 3 primary advantages of a serverless architecture over a traditional one are cost, scalability, and availability.

Sounds great, but how does that apply to my company?

One of the most popular use cases for serverless is APIs. Using AWS DynamoDB, Lambda, and API Gateway allows us to create blazingly fast and scalable applications for a fraction of the cost of more traditional approaches. And let's be honest, every single web application can benefit from that.

This was a passion project developed by Turner over the course of 3 months. The front end is a single page application built in VueJS, hosted on an AWS S3 bucket and features a responsive design.

A challenging part of this project was implementing Websockets on the front end for real-time updates. After much research and experimentation, he found that the AWS IoT (Internet of Things) service has a message broker that works perfectly as an MQTT client if configured in the right way. This allows the application to send notifications to users when a message is sent, or a chat invite is accepted.

Overall, this was a great experience for Turner and hopes to use this as a backbone of knowledge for future appliations. If you want to learn more about the application, you can go to the about page on the site.

Xource API

This is a project that is still in development. It is all configured to run inside an AWS VPC (virtual private cloud) and accesses an AWS Aurora MySQL database provided on AWS RDS, which is connected to by Lambda and API Gateway. This allows us to use a relational database for application storage instead of a document-based one.

This has been a challenging but interesting project for Turner as he is responsible for creating and maintaining all of the serverless infrastructure for the team as well the deployment pipeline for doing so, which uses a combination of AWS CodeCommit, CodePipeline, and CodeBuild.
