Why Serverless?

Serverless is the latest back-end architecture, born out of services like AWS Lambda. This allows for highly scalable and low cost applications that can be run on large server clusters, like the ones provided by Amazon Web Services. The 3 primary advantages of a serverless architecture over a traditional one are speed, cost, and scalability.

Sounds great, but how does that apply to me?

One of the most popular use cases for serverless is APIs. Using AWS DynamoDB, Lambda, and API Gateway allows us to create blazingly fast and scalable applications for a fraction of the cost of more traditional approaches. And let's be honest, every single web application can benefit from that.

Are there any examples of applications that use this style of architecture?

AWS Chat

This was a passion project developed by Turner over the course of 3 months. The front end is a single page application built in VueJS, hosted on an AWS S3 bucket and features a responsive design. The back end is a serverless API, built with AWS services including Lambda, Cognito, API Gateway, DynamoDB, and Internet of Things. It is deployed using the Serverless Framework.

A challenging part of this project was implementing Websockets on the front end for real-time updates. After much research and experimentation, he found that the AWS IoT (Internet of Things) service has a message broker that works perfectly as an MQTT client if configured in the right way. This allows the application to send notifications to users when a message is sent, or a chat invite is accepted.

Overall, this was a great experience for Turner and hopes to use this as a backbone of knowledge for future appliations. If you want to learn more about the application, go to awschat.net/about

Xource API

This is a project that is still in development. It is all configured to run inside an AWS VPC (virtual private cloud) and accesses an AWS Aurora MySQL database provided on AWS RDS, which is connected to by Lambda and API Gateway.

This has been a challenging but interesting project for Turner as he is responsible for creating and maintaining all of the serverless infrastructure for the team as well the deployment pipeline for doing so, which uses a combination of AWS CodeCommit, CodePipeline, and CodeBuild.
